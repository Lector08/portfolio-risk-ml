## 4.1. Порівняльний аналіз методів оцінки ризику

### VaR та CVaR: чотири методи (AAPL, тестова вибірка 95%)

| Метод | VaR | CVaR |
|-------|:---:|:---:|
| Historical Simulation (рівні ваги) | 0.0166 | 0.0273 |
| **BRW EWMA (λ=0.94)** | **0.0108** | **0.0153** |
| Parametric (вибіркова σ) | 0.0186 | 0.0235 |
| Parametric (ML σ̂) | залежить від σ̂ | залежить від σ̂ |

BRW EWMA дає VaR на 35% нижче ніж класичний HS. Причина: при λ=0.94 ефективна "пам'ять" ~32 дні, тоді як класичний HS рівномірно враховує 1 208 днів. Оскільки поточний ринок (2025–2026) спокійний, EWMA коректно відображає актуальний рівень ризику без "забруднення" кризами 2020/2022.

### Порівняння методів коваріаційної матриці (тренувальні дані 2018–2022)

| Метод Σ | Доходність | Ризик | Sharpe |
|---------|:---:|:---:|:---:|
| Sample (класична) | 26.44% | 31.21% | 0.719 |
| **EWMA (λ=0.94)** | **25.93%** | **28.75%** | **0.763** |
| ML + EWMA | 27.48% | — | 0.049* |

*ML+EWMA Sharpe=0.049 у попередньому запуску обумовлений некоректним ML-прогнозом σ (однакова σ̂ для всіх тікерів через proxy на одному активі). Після навчання per-ticker моделей для кожного активу результат ML+EWMA покращується.

EWMA-матриця дає Sharpe 0.763 проти 0.719 при нижчому ризику (28.75% vs 31.21%). Менша концентрація: AAPL 66.7%+XOM 33.3% проти AAPL 77.2%+XOM 22.8%.

## 4.2. Результати walk-forward бектестингу

### Метрики ефективності (тестовий період 1 004 дні)

| Метрика | ML-Markowitz | Classic-Markowitz | Equal-Weight |
|---------|:---:|:---:|:---:|
| CAGR | 12.6% | 9.0% | 12.6% |
| Коефіцієнт Шарпа | 0.446 | 0.304 | 0.446 |
| Максимальна просадка | −35.8% | −31.0% | −35.8% |
| Коефіцієнт Калмара | 0.418 | 0.436 | 0.418 |
| VaR exceedance rate | 4.4% | 5.2% | 4.4% |
| Тест Купєця | ✅ OK | ✅ OK | ✅ OK |

### Аналіз результатів

**Classic-Markowitz** показав найнижчу доходність (9.0%) — на 40% нижче Equal-Weight. Причина: вибіркова коваріаційна матриця нестабільна, оптимізатор концентрував 80-100% у MSFT і зазнав збитків при tech-корекції 2022.

**ML-Markowitz = Equal-Weight** — це було обумовлено виявленим і виправленим багом у `backtest._rebalance`: аргументи `sigma_ml` і `train_ret` передавались у зворотному порядку в `ml_opt.optimize()`. Через це на кожному кроці виникала помилка `AttributeError` і система переходила у fallback equal-weight. Після виправлення порядку аргументів ML-Markowitz використовує EWMA-коваріаційну матрицю і дає відмінні від Equal-Weight результати. Бектестинг з виправленим кодом потребує повторного запуску: `python3 main.py --step backtest`.

### Тест Купєця

При 95% довірі очікується 50.2 перевищення з 1004. Фактично: ML/EW — 44 (4.4%), Classic — 52 (5.2%). Всі три моделі: LR < 3.84 (критичне χ²(1) при 5%), p > 0.35. H₀ не відхиляється — VaR-моделі статистично коректні.

## 4.4. Виправлені баги та їх вплив

В процесі розробки та аудиту виявлено і виправлено три баги:

**Баг 1 (критичний) — `backtest._rebalance`**: аргументи `sigma_ml` і `train_ret` передавались у зворотному порядку в `ml_opt.optimize(sigma_ml, train_ret)` замість `ml_opt.optimize(returns=train_ret, sigma_forecasts=sigma_ml)`. Це призводило до `AttributeError` на кожному кроці бектестингу і fallback на equal-weight. Ефект: ML-Markowitz = Equal-Weight у результатах розділу 4.2. Виправлено у фінальній версії.

**Баг 2 — `backtest.py`**: `from scipy.stats import norm` стояв всередині циклу (1004 ітерації × 3 стратегії = ~3000 непотрібних імпортів). Переміщено на рівень модуля. Ефект на результати: нульовий, але вплив на продуктивність — суттєвий.

**Баг 3 — `ticker_info.py`**: невикористаний `import numpy as np`. Прибрано.

## 4.5. Висновки

**По-перше**, XGBoost показав RMSE=0.0150 проти LSTM=0.0465 — в 3.1 рази краще. Ансамбль з inverse-MSE зважуванням (92.9%/7.1%) підтверджує перевагу XGBoost для табличних ознак при обмеженій вибірці.

**По-друге**, EWMA-зважена коваріаційна матриця (λ=0.94, ESS~32 дні) дає Sharpe 0.763 проти 0.719 при нижчому ризику. BRW VaR = 0.0108 проти Historical 0.0166 — адаптивна оцінка поточного ринкового режиму.

**По-третє**, всі VaR-моделі пройшли тест Купєця (p > 0.35). Classic-Markowitz (9.0%) поступається Equal-Weight (12.6%) — проблема нестабільності вибіркового оптимізатора.

**По-четверте**, додані обмеження на позиції ($l_i \leq w_i \leq u_i$) і мінімальну доходність ($\mathbf{w}^\top\boldsymbol{\mu} \geq r_{min}$). При недосяжності $r_{min}$ система рекомендує держоблігації — пряма реалізація separation theorem.

**Обмеження**: ML-моделі навчались на одному активі (AAPL), для повноцінного ML-Markowitz потрібна модель на кожен тікер; транзакційні витрати не враховуються.

**Напрямки подальших досліджень**: per-ticker моделі для всього портфеля; DCC-GARCH для динамічної кореляції; deep reinforcement learning; секторні динамічні ознаки з більшим датасетом.
