## 3.1. Архітектура системи та стек технологій

Система управління ризиками портфеля реалізована як модульний Python-застосунок із чіткою відповідністю між функціональними блоками та файлами проекту. Пайплайн складається з чотирьох шарів: завантаження та обробка даних, ML-моделювання волатильності, розрахунок ризик-метрик та оптимізація портфеля, бектестинг та візуалізація.

Технологічний стек: Python 3.11, yfinance 0.2+ для отримання ринкових даних, pandas/numpy для числових обчислень, TensorFlow 2.15 (Keras) для LSTM, XGBoost 2.0 для градієнтного бустингу, scikit-learn для передобробки, cvxpy для опуклої оптимізації, arch для GARCH-базлайну, matplotlib/seaborn для візуалізації.

## 3.2. Завантаження та обробка ринкових даних (M1)

### Вибірка даних

Для дослідження обрано 8 акцій S&P 500, що представляють різні сектори економіки: технологічний (AAPL, MSFT, GOOGL), споживчий (AMZN), фінансовий (JPM, GS), енергетичний (XOM) та охорони здоров'я (JNJ). Часовий діапазон — 2018–2024, що включає 1 508 торгових днів та охоплює різні ринкові режими: бичачий ринок 2019–2020, COVID-крах березня 2020, відновлення 2020–2021, корекцію 2022 року.

### Завантаження та кешування

Модуль `data/loader.py` реалізує завантаження скоригованих цін закриття (`auto_adjust=True`) через бібліотеку yfinance. Скориговані ціни враховують дивіденди та спліти акцій, що забезпечує коректне порівняння цін у часі. Для усунення пропущених значень (вихідні, свята різних ринків) застосовується forward-fill з обмеженням у 2 дні, після чого рядки з залишковими NaN видаляються.

Реалізовано систему кешування: після першого завантаження дані зберігаються у `data/raw/prices.csv`, і при наступних запусках система зчитує локальний файл без звернення до API.

### Логарифмічні доходності

З цін закриття обчислюються логарифмічні доходності:
$$r_t = \ln\left(\frac{P_t}{P_{t-1}}\right)$$

Перевага логарифмічних доходностей перед арифметичними ($r_t = P_t/P_{t-1} - 1$): адитивність у часі ($r_{0\to2} = r_{0\to1} + r_{1\to2}$), симетрія зростання та падіння, краща апроксимація нормальним розподілом. Остання властивість є важливою передумовою для параметричного VaR.

### Цільова змінна: реалізована волатильність

Для навчання ML-моделей прогнозу волатильності використовується реалізована волатильність (Realized Volatility, RV) — стандартний проксі "істинної" волатильності в академічній літературі:
$$\text{RV}_t = \sqrt{\sum_{i=t-w+1}^{t} r_i^2} \cdot \sqrt{\frac{252}{w}}$$

де $w = 20$ (торгових днів, ~1 місяць). Множник $\sqrt{252/w}$ анулізує волатильність для зручності порівняння.

### Часовий розподіл даних

Дані розподіляються хронологічно у співвідношенні 80/10/10:

| Вибірка | Період | Рядків |
|---------|--------|--------|
| Train | 2018-01-02 – 2021-10-31 | 1 207 |
| Validation | 2021-11-01 – 2022-11-30 | 151 |
| Test | 2022-12-01 – 2023-12-31 | 150 |

Перемішування рядків (shuffle) категорично неприпустиме для часових рядів, оскільки призводить до data leakage: модель "бачить майбутнє" під час навчання, а результати на тесті є завищеними і не відображають реальну якість.

### Feature Engineering

Для XGBoost-моделі будується матриця ознак із 4 груп: ковзні середні (MA ratio для вікон 5 та 20 днів), RSI(14) нормалізований до [-1, 1], смуги Боллінджера (%B та bandwidth), rolling-статистики (std, skewness, kurtosis для вікон 5/10/20 днів) та лагові доходності (лаги 1, 2, 3, 5, 10). Загалом матриця ознак містить **80 ознак** для 8 тикерів.

Для LSTM-моделі з матриці ознак формується 3D-тензор `(samples, 60, 80)`, де 60 — розмір ковзного вікна (останні 60 торгових днів). Це дозволяє моделі явно використовувати часову структуру даних.
