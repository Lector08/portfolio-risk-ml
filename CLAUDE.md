## Алгоритм роботи (флоу)

### Порядок написання
1. **Розділи 1–2 звіту** — пишуться заздалегідь, до коду. Можуть оновлюватись.
2. **По кожному Milestone** — пишемо код, потім одразу описуємо його в розділі 3.
3. **Розділ 4** — пишеться останнім, після завершення M5 (бектестинг).

### Правила оновлення
- Розділи 1–2: написані, але **відкриті до уточнень** в процесі роботи.
- Розділ 3: кожен підрозділ (3.2, 3.3, ...) пишеться **одразу після** відповідного Milestone.
- Розділ 4: **тільки після** отримання фінальних результатів бектестингу.

## Правила стилю коду

- **Docstrings** — обов'язкові для кожної функції та класу
- **Type hints** — для всіх аргументів і return values
- **Коментарі** — українською мовою
- **Константи** — виключно з `config.py`, не хардкодити
- **Часові ряди** — НІКОЛИ не перемішувати (data leakage!)
- **Train/Val/Test** — хронологічний split 80/10/10
- **Збереження моделей** — XGBoost у `.json`, LSTM у `.keras`

## Ключові математичні об'єкти

```
r_t = ln(P_t / P_{t-1})                         ← лог-доходність
RV_t = sqrt(sum r_i^2) * sqrt(252/w)             ← реалізована волатильність (ціль ML)

VaR_α = -(μ + σ * z_α)                          ← параметричний VaR
CVaR_α = -E[r | r ≤ -VaR_α]                     ← Expected Shortfall

min w^T Σ w,  s.t. w^T μ ≥ r*, 1^T w = 1, w ≥ 0 ← задача Марковіца
Σ_ML = D · Corr · D,  D = diag(σ̂_1,...,σ̂_n)   ← ML-коваріаційна матриця

w_i = (1/MSE_i) / Σ(1/MSE_j)                    ← ваги ансамблю
S = (μ_p - r_f) / σ_p                           ← коефіцієнт Шарпа
```
